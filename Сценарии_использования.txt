Идентификатор и название : UC-1 Войти в личный кабинет
Основное действующие лицо: Пешеход, Автомобилист, Пользователь общественного транспорта, Водитель общественного транспорта
Описание: Пользователь вводит свои персональные данные (логин и пароль). Система обрабатывает полученные данные и предоставляет доступ к системе или отказывает в нем.
Триггер: Пользователь хочет войти в личный кабинет

Предварительные условия:
PRE-1 База данных, которая хранит информацию о пользователях, должна быть доступна.
PRE-2 Заполнены поля для входа в личный кабинет (логин и пароль).

Выходные условия:
POST-1 Запрос отправлен к базе данных, которая хранит информацию о пользователях.
POST-2 Успешная авторизация в системе, отбражена карта города.

Нормальное направление развития варианта использования: 
1.0 Все данные заполнены и введены верно
1. Пользователь вводит свои данные.
2. Система отправляет запрос к базе данных о существовании учетной записи.
3. Система отображает результат запроса.
5. Пользователь авторизовался в системе.
5. Система отображает карту города.

Альтернативное направление развития варианта использования: 
1.1 Пользователь ввел данные с ошибкой.
1. Пользователь неправильно ввел данные.
2. Система отправляет запрос на существование учетной записи.
3. Система отображает сообщение "Неправильно введен логин или(и) пароль".
4.а Пользователь исправляет свои данные.
5. Возврат к п. 2 нормального направления.
4.б Пользователь решает выйти из системы.

Исключения:
1.0.Е1 Пользователя нет в базе данных
1. Система отображает сообщение "Пользователь с такими данными не найден"
2. Система выводит дополнительное окно, в котором предлагает регистацию для нового пользователя, или предлагает выйти из системы
3.а Пользователь вводит данные для регистрации
4.а Данные передаются в БД 
5.а Система заново начинает нормальное направление варианта использования
3.б Пользователь решает выйти из системы
4.б Система завершает вариант использования
Приоритет: Высокий 
Частота использования: Каждый раз при запуске приложения

Идентификатор и название : UC-2 Отобразить карту
Основное действующие лицо: Пешеход, Автомобилист, Пользователь общественного транспорта, Водитель общественного транспорта
Описание: Пользователь открывает приложение, вводит свои данные, авторизовывается и просматривает карту, в целях определения наиболее удобного маршрута. 
Триггер: Авторизация в системе.
Предварительные условия:
PRE-1 Должно быть доступно подключение к серверу для получения данных.
PRE-2 Пользователь успешно авторизован.

Выходные условия:
POST-1 Отправка запроса к серверу для получения информации по расположению объектов в городе. 
POST-2 Система отображает карту для конкретного класса пользователя.
POST-3 Система обновляет информацию о количестве определенного класса пользователей на маршрутах.

Нормальное направление развития варианта использования: 
2.0 Отображение карты на основе личных данных
1. Система определяет класс пользователя и его местоположение.
2. Система отображает карту города.
3. Пользователь просматривает маршруты.
4. Пользователь выбирает маршрут или изменяет класс пользователя( обратно к п.1)
5. Система отображает маршрут и расчитывает время до прибытия. 
6. Система сохраняет выбранный маршрут в базе данных, добавляет маршрут в личный кабинет и обновляет информацию о загруженности дорог.

Альтернативное направление развития варианта использования:
2.1 Класс пользователя неуказан
1. Система определяет местоположение пользователя.
2. Система определяет класс "Пользователь общественного транспорта".
3. Возврат к п.2 нормального направления.

Исключения:
2.0.Е1 Проблемы с подключением к серверу.
1. Система отображает сообщение "Нет соединения с сервером".
2. Система предложит перейти в личный кабинет или закрыть приложение.
3.а Система предложит оставить отзыв.
3.б Пользователь решает выйти из системы.
4.б Система завершает вариант использования.
Приеоритет: Высокий 
Частота использования: Каждый раз при открытии приложения
